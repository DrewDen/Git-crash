Branches

This section will cover branches. Branches are great to use when colaborating with a other developer on a project. This gives you chance to leave the main source code alone so you can work on a copy of it.

Here are some steps to go by when working with branches

  • To set up a branch us the command git branch [name of new branch] this will set up a new branch.
  • After you set that up you can switch to it using the command git checkout [branch name] and this will switch to that branch.
  • You will now be on a seprate branch from the master. You can add any code you want into the new branch.
  • If you want to switch back into the master branch you can you use the command git checkout master. This will switch you back into the master branch and you will see any code you created on the other branch removed.
  • If you looking to add the two branches together you can use a command called git merge and it will combined the branches together.
  • When you use merge yoo have to be on the branch where you want your other branch to merge into.

    • example I want branch login to merge in with the master branch. I would need to be on the master branch for this to happen: on master branch git merge login.
